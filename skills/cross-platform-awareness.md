# Cross-Platform Awareness

Both apps depend on the same Rust core and should stay aligned on shared product behavior.

## Rules

1. Core changes affect both platforms.
   After modifying `core/`, run:
   ```bash
   just generate
   just ios build
   just android build
   ```
2. Feature parity matters.
   If a user-facing change should exist on both platforms, either implement both sides or call out the gap explicitly.
3. Shared models are generated — never edit them directly.
   Files marked `Generated by typeshare` come from `core/`. Generated file locations:
   - **iOS**: `Packages/Primitives/Sources/` (TypeShare models), `GemstoneFFI.xcframework` (UniFFI binary)
   - **Android**: `gemcore/` (JNI bindings), generated Kotlin models under `data/` modules
   - **Pattern**: any file with a `// Generated by typeshare` header or under `uniffi/gemstone/`
4. Localization is shared.
   Run `just localize` when changing shared localization inputs.
